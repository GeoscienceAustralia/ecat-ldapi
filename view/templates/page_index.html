Dataset Register Linked Data API

Home

This Linked Data API presents multiple model views and formats for Geoscience Australia's datasets as managed within our eCat Catalogue. This system is currently in test and most of the views below are incomplete.

For the authoritative listing of GA's datasets and publications, please see eCat:

  • eCat Catalogue

Documentation

The way this Linked Data API works and the various endpoints for it are described on the Documentation and API pages, linked to above. Also, there are a series of example endpoints with descriptions given below.

For further information, contact the GA team that manage this system via the About page.

Examples

To use this Linked Data API, follow the links below for examples of different model views and formats of the objects it delivers:

  • Dataset 76068 - a single Dataset, Isostatic Residual Gravity Anomaly Grid of Onshore Australia - 2011
    • ISO19115-1:2014 view, in HTML
      • This web page view of the deataset's metadata, according to the ISO19115-1:2014 Geographic Information -- Metadata schema redirects, to the dataset's homepage in eCat where the view is provided by the GeoNetwork catalogue software that GA uses.
    • ISO19115:2014-1 view, in XML
      • This is the machine-readable, XML format of the ISO19115:2014 Geographic Information -- Metadata schema, serialised according to the ISO19115-3:2016 Geographic information -- Metadata -- Part 3: XML schema implementation for fundamental concepts. This view & format is also provided by eCat's GeoNetwork catalogue software.
    • Alternates view, in HTML
      • This is the view that lists all other views and formats. This view is available in HTML (for humans) and a range of machine-readable formats and includes an extended version of this list of views and formats. An alternates view is available for every Linked Data class (here: Dataset & Register).
    • AGRIF view, in RDF (turtle)
      • This is the view of this dataset as a Record, accrding to the AGRIF ontology. It is delivered in machine-readable RDF formats only.
    • AGLS view, in XML
      • This is the National Archive's AGLS view of the dataset. AGLS is "improve visibility and availability of online resources". This view is essentially Dublin Core Terms with some additions. It is available in XML and RDF machine-readable formats only.
  • Register - the list of datasets
    • Registry view, in HTML
      • A basic web page list of the items in this register (Datasets) with information about how to page through the list.
    • Registry view, in RDF (turtle)
      • A simple list view of the Datasets Register according to the Registry ontology. A very simple of a register with contained items. Useful for Linked Data applications to find all the individuals in a register. Can be paginated by a machine.
    • Alternates view, in HTML
      • This is the view that lists all other views and formats.
